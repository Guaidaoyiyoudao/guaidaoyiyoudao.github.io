ELF文件结构

ELF header

e_ident

指定了如何去解释该文件。

0-3四个字节的内容是一个魔数，内容为0x7f,E,L,F。

EI_CLASS

第五个字节，显示二进制文件的架构：

01表示是32位，02表示是64位。

EI_DATA

第六个字节显示是大端还是小端:

01 小端，02大端。

大端模式，是指数据的高字节保存在内存的低地址中，而数据的低字节保存在内存的高地址中，这样的存储模式有点儿类似于把数据当作字符串顺序处理：地址由小向大增加，而数据从高位往低位放；这和我们的阅读习惯一致。
小端模式，是指数据的高字节保存在内存的高地址中，而数据的低字节保存在内存的低地址中，这种存储模式将地址的高低和数据位权有效地结合起来，高地址部分权值高，低地址部分权值低。

EI_VERSION

第七个字节，显示版本号：

01当前版本

EI_OSABI

第八个字节，选择的操作系统和ABI版本:

EI_ABIVERSION 第九个字节，ABI版本号,用来辨别ABI版本。这个的值取依赖于EI_OSABI

EI_PAD 填充开始，通常值被设为0。应用程序应该忽略这些值，这个区域可能在以后会有其他的用途，目前无用。

EI_NIDENT e_ident的数组的大小，从上面的定义看是16个字节。

e_type

17-18字节

指定了文件的类型

e_machine

19-20字节

对于单个文件所要求的架构

e_version

20-24 字节

文件版本号

e_entry

25-32字节
32位系统就是4个字节大小

给出了程序的入口点，没有的话就是0

e_phoff

33-40字节
32位系统就是4个字节大小
程序头的字节偏移量

e_shoff

41-48字节
32位系统就是4个字节大小
节区头的字节偏移量

e_flags

49-52字节
目前未定义

e_ehsize

53-54 字节
elf header的大小

e_phentsize

55-56字节
程序头在程序头表中的大小，所有的程序头都是同样大小。

e_phnum

57-58字节
程序头的数量

e_shentsize

59-60字节
节区头的大小

e_shnum

61-62字节
节区头的数量

e_shstrndx

63-64字节

This member holds the section header table index of the entry associated with the section name string table. If the file has no section name string table, this member holds the value SHN_UNDEF.

程序头

程序头只对可执行文件和共享目标文件有效。

p_type

PT_PHDR

定义文件头表在文件和内存image的位置和大小，只会有一个这样类型的文件头

PT_GNU_STACK

通过p_flags来控制栈的状态

p_flags

PF_X An executable segment.
PF_W A writable segment.
PF_R A readable segment.

p_offset

从文件开始到首个字节出现的偏移量。

section header

.bss

保存了未初始化的数据，定义这个section,当程序运行的时候，会把数据全部初始化为0

.data

保存了初始化的数据

.text

保存了可执行指令

.rodata

保存了只读数据
