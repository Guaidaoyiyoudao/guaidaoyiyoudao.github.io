lavarel学习记录

Posted on 2019-04-27 | In php |

文件目录解释

App

该目录包含了应用的核心代码，大多数类都在这个目录里面。

Bootstrap

Bootstrap里面的app.php用来启动框架。
cache目录包含性能优化的文件。

Config

应用所有的配置文件都在该目录，需要熟悉里面的选项。

Database

包含了factories，migrations,seeds,如果想使用sqlite数据库，可以把文件放在这个目录里面。

Public

里面index.php是所有应用请求和自动加载配置的入口点。这个目录也包括了图片，css,以及javascript。

Routes

包含了应用的所有路由的定义，默认的几个路由文件是:web.php,api.php,console.php,channels.php。

web.php:如果应用不提供无状态,RESTful的api的话，所有的路由定义基本都在这个文件里面,提供会话状态，CSRF防护，cookie加密。

channels.php: 所有的注册的事件广播。

Storage

包含了编译的Blade模板，会话文件，文件缓存，以及其他的框架产生的文件。
该目录下的每个目录对应不用的模块产生的文件的分类，如log就是应用的日志文件。

命令

php artisan list make 列出所有的Artisan commands,通过这个可以查看怎么生成默认没有的目录。

App目录解释

http

包含了控制器，中间件以及表单请求。大多数处理应用请求的逻辑都在这个目录里。

Exceptions

用来处理应用抛出的异常。

请求的生命周期

首先

Lavarel应用对于所有请求的入口点是文件public/index.php，所有的请求由web server发送给这个文件。index.php没有包含很多的代码，这个文件只是加载框架的起始点。

Http / Console Kernels

然后，请求会根据请求的类型发送给http kernel 或者 console kernel。

htpp kernel文件在该目录下:app/http/kernel.php

http kernel包含了bootstrapers，这些bootstrapers会配置错误处理，日志，检测应用环境，以及执行其他需要在请求执行之前完成的一些任务。

http kernel也定义了一些中间件，负责读和写http session,决定应用程序是否在 maintenance mode，验证CSRF token…

htpp kernel里面的方法handle非常简单，即接受一个请求，返回一个响应。

# php # lavarel
